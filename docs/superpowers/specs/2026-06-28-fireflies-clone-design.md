# Fireflies.ai Clone — Design Spec

**Date:** 2026-06-28
**Status:** Approved for implementation

## 1. Overview

A functional clone of the Fireflies.ai meeting-assistant web app. Users browse a
library of meetings, view interactive transcripts with speaker labels and
timestamps, read AI-generated summaries / action items / key topics, search
across transcripts, and manage meetings (CRUD). Real speech-to-text is out of
scope; transcripts are seeded, uploaded (`.txt`/`.vtt`/`.json`), or pasted, and
summaries are seeded or generated by a deterministic rule-based generator.

## 2. Decisions (locked)

- **AI generation:** Seeded data for samples + rule-based mock generator for new
  uploads. No external API key required; demo works fully offline and free.
- **Scope:** All core features + selected high-value bonuses (global search,
  export to Markdown/TXT, dark mode, tags + filtering). Other bonuses stubbed.
- **Styling:** Tailwind CSS.
- **Deployment:** Deployment-agnostic — Docker + docker-compose + clear README;
  host chosen later (Vercel/Render/Railway all viable).
- **Media player:** Bundled sample audio file with a real working seek bar,
  two-way synced to the transcript (click line → seek; playback → highlight).

## 3. Architecture

```
fireflies-clone/
├── backend/   FastAPI + SQLAlchemy + SQLite
│   └── app/{main,database,models,schemas,crud,seed}.py
│       routers/{meetings,action_items,search}.py
│       services/{generator,parser}.py
├── frontend/  Next.js (App Router) + TypeScript + Tailwind
│   ├── app/{page,layout}.tsx  app/meetings/[id]/page.tsx  app/settings/page.tsx
│   ├── components/  (cards, panels, player, modals, toasts, search)
│   └── lib/api.ts
├── docker-compose.yml
└── README.md
```

- **Backend layering:** routers → crud → models; services for parsing/generation.
  Pydantic v2 schemas for request/response. SQLite file DB.
- **Frontend:** App Router, typed API client, React Context for theme (dark mode)
  and toasts. Default logged-in user (no real auth).

## 4. Database Schema

- **Meeting**: id, title, description, date, duration_seconds, audio_url,
  created_at, updated_at.
- **Participant**: id, meeting_id (FK), name, email, role.
- **TranscriptSegment**: id, meeting_id (FK), speaker, start_ms, end_ms, text,
  order_index. `start_ms` drives seek + highlight.
- **Summary** (1:1): id, meeting_id (FK unique), overview, generated_by.
- **KeyTopic/Chapter**: id, meeting_id (FK), title, start_ms, order_index.
- **ActionItem**: id, meeting_id (FK), text, assignee, due_date, completed,
  order_index, created_at.
- **Tag** + **meeting_tags** (M2M).

Relationships: Meeting 1→N Participant/TranscriptSegment/KeyTopic/ActionItem,
1→1 Summary, M2M Tag. Cascade-delete children on meeting delete.

## 5. API (REST, prefix `/api`)

| Method | Path | Purpose |
|---|---|---|
| GET | `/meetings` | list; `?search=&participant=&tag=&sort=recent` |
| POST | `/meetings` | create (metadata form, or upload/paste transcript) |
| GET | `/meetings/{id}` | full detail |
| PATCH | `/meetings/{id}` | edit metadata/participants |
| DELETE | `/meetings/{id}` | delete + cascade |
| POST | `/meetings/{id}/transcript` | upload/paste transcript → parse → regenerate |
| POST | `/meetings/{id}/action-items` | add action item |
| PATCH | `/action-items/{id}` | edit / toggle complete |
| DELETE | `/action-items/{id}` | delete |
| GET | `/search?q=` | global search across all transcripts |

## 6. Services

- **parser.py**: parse `.txt` (Speaker: text lines, optional `[mm:ss]`), `.vtt`
  (cues → segments), `.json` (segment array). Produces ordered segments with
  timestamps; synthesizes timestamps when absent.
- **generator.py** (rule-based, deterministic):
  - Summary overview: top sentences by keyword frequency.
  - Action items: sentences matching action cues ("will", "need to", "TODO",
    "let's", "action", "follow up", "by <date>"), with assignee heuristic.
  - Key topics/chapters: keyword clustering across segments with start times.

## 7. Feature → Implementation map

- **Dashboard:** card grid, search, participant/tag filters, sort-by-recency,
  sidebar + navbar.
- **Detail:** transcript panel (speaker/timestamp lines, in-transcript search +
  highlight), summary/action-items/topics tabs, media player with two-way sync.
- **CRUD:** create (form/upload/paste), edit, delete, action-item CRUD + toggle.
- **Persistence:** all entities in SQLite.
- **Bonuses included:** global search, export (Markdown/TXT), dark mode, tags.
- **Stubbed "Coming Soon":** live bot, real STT, integrations, team/sharing, auth.

## 8. Seed Data

Several full meetings (varied titles/participants/durations) each with complete
transcripts (multi-speaker, timestamped), seeded summaries, key topics, and
action items, so the app is immediately usable on first run.

## 9. Out of Scope

Real audio transcription, live meeting bot, third-party integrations, real
authentication, team collaboration — all present as "Coming Soon" placeholders.
